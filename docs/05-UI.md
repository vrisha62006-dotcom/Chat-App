# User Interface (UI) Design Spec - ChatConnect

This document describes the structure, elements, and layouts of the ChatConnect client screens.

---

## 1. Authentication Screens

### 1.1 Login Page
* **Visual Structure:** A centered, glassmorphic card on a dark background gradient.
* **UI Elements:**
  * Application Logo / Brand Name header ("ChatConnect").
  * Input field for `Username`.
  * Input field for `Password` (masked text input).
  * "Sign In" CTA button (accent color with hover effects).
  * Toggle link: "Don't have an account? Sign Up".
* **Validation Behavior:** Displays inline error states for empty values or failed credential matches.

### 1.2 Register Page
* **Visual Structure:** Matches the visual layout of the login screen for style consistency.
* **UI Elements:**
  * Header label ("Create an Account").
  * Input field: `Username` (restricted to alphanumeric characters).
  * Input field: `Password` (requirements: min 6 characters).
  * Input field: `Confirm Password`.
  * "Register Account" CTA button.
  * Toggle link: "Already registered? Sign In".

## 2. Main Chat Workspace Layout
The main interface uses a split-screen viewport layout that adjusts for screen widths.

```text
+-----------------------------------------------------------------------+
|  ChatConnect                      [User Profile Icon] [Logout Button] |
+-------------------------------------------------+---------------------+
| SIDEBAR                                         | CHAT WINDOW         |
| [Search Bar: Find users/groups]                | [Room Header: Name] |
|                                                 | [Online status indicator]
| --- CHATS LIST ---                              +---------------------+
| (o) Alice  "Sure, talk later"     [10:15 AM]    | Alice: Hi there!    |
| (x) Bob    "Sent a group link"    [Yesterday]   |                     |
| (#) Design Team (3 members)       [Friday]      | You: Hello Alice    |
|                                                 |                     |
|                                                 | [Typing status...]  |
|                                                 +---------------------+
| [Button: Create Group]                          | [Input Box...] [Send]|
+-------------------------------------------------+---------------------+
```

### 2.1 Sidebar Panel (Left Pane)
* **Header Components:** Logo branding and User Profile control options.
* **Search / Filter Control:** A quick search box to dynamically filter existing rooms or search the global user directory to initiate direct chats.
* **Rooms & Chats Lists:** Divided into "Direct Chats" and "Group Chats". List items display avatars, presence markers (green for online, grey for offline), truncated latest message strings, and timestamps.
* **Group Creation Action:** A button at the bottom of the sidebar to initiate group creation.

### 2.2 Chat Window (Right Pane)
* **Header Area:** Active chat target name (Username or Group name), presence state description (e.g. "Online" or "3 Members"), and configuration options (e.g., View Group Members, Leave Group).
* **Messages Log Feed:** Vertical scroll list showing message cards.
  * Received messages are aligned to the left, styled in a dark grey card.
  * Sent messages are aligned to the right, styled in an accent-colored card.
  * Message bubbles include the body text and small, aligned timestamps at the bottom.
* **Typing Indicator Bar:** Appears right above the message input field (e.g. "Bob is typing...") when typing socket events are received.
* **Message Input Control Area:**
  * Auto-growing text area field.
  * "Send" action button (represented by a paper plane icon).

## 3. Modals & Dialogs

### 3.1 Create Group Dialog
* **Visual Structure:** A modal overlay that dims the main application workspace.
* **UI Elements:**
  * Header title ("Create New Group").
  * Input field: `Group Name`.
  * Multi-select contacts picker: A list of users with checkboxes to add to the group.
  * Action Buttons: "Cancel" (neutral outline style) and "Create Group" (accent colors).

### 3.2 User Profile Modal
* **UI Elements:**
  * Displays user profile attributes (Avatar, Username, Account registration date).
  * Active websocket status indicator.
  * Action Button: "Close".

---

## 4. Mobile Responsive Layout Rules
To provide a premium experience on mobile, ChatConnect implements a adaptive breakpoint layout:

* **Breakpoint (max-width: 768px):**
  * The interface switches from a two-pane layout to a single-pane layout.
  * By default, the application displays the **Sidebar** list of conversations.
  * Selecting a conversation slides the **Chat Window** into full view, covering the Sidebar.
  * The Chat Window header includes a back button (`<-`) to return to the Sidebar.
